01

Contraste de color

El contraste de color es la falla de accesibilidad más común. WCAG 2.1 requiere 4.5:1 para texto normal (AA) y 7:1 para nivel mejorado (AAA). El ratio 3:1 aplica a texto grande (≥18pt) y componentes UI.

Presets rápidos

El diseño accesible beneficia a todos. Los textos con bajo contraste son difíciles de leer bajo luz solar directa, para personas con baja visión, o con ojos cansados. No es solo para usuarios con discapacidad.

— ratio de contraste

Tip: el gris claro sobre blanco es la falla más común. Si dudás, oscurecé el texto.

02

Áreas táctiles

WCAG 2.5.5 (AAA) requiere que los elementos interactivos sean de al menos 44×44 px. WCAG 2.5.8 (AA, versión 2.2) pide 24px. Objetivos pequeños causan errores, especialmente en dispositivos táctiles o con temblor.

32px

Verde = cumple WCAG (≥44px) · Rojo = no cumple. Probá con el tamaño más chico.

03

Tipografía legible

WCAG 1.4.4 requiere que el texto pueda escalarse al 200% sin pérdida de contenido. WCAG 1.4.8 establece guías de interlineado, ancho de columna y espaciado. Una tipografía accesible mejora la lectura para todos.

OK
OK
OK

El cuerpo del texto debe ser legible en condiciones reales de uso. La tipografía accesible no es una opción estética, es un requisito funcional. Un buen tamaño, interlineado generoso y columna de ancho moderado reducen la carga cognitiva y mejoran la comprensión.

04

Lectores de pantalla

Los lectores de pantalla anuncian elementos usando texto alternativo, labels y HTML semántico. La diferencia entre un label vago y uno descriptivo es la diferencia entre confusión y claridad.

Malo Botón sin descripción
El lector anuncia: botón

Sin aria-label, el lector anuncia solo "botón". ¿Botón de qué? El usuario no puede saber.

Bueno Botón con aria-label
El lector anuncia: Ir al carrito de compras, 3 artículos

aria-label describe la acción específica. El usuario entiende el propósito sin ver la pantalla.

Malo Imagen sin alt
IMG
El lector anuncia: imagen sin descripción

Sin atributo alt, el lector omite la imagen o lee el nombre del archivo. Información perdida.

Bueno Imagen con alt descriptivo
IMG
El lector anuncia: Fotografía de un escritorio minimalista con laptop, libreta y café

alt describe el contenido, no "imagen de" (eso es redundante). Incluir la información que aporta la imagen.

05

Navegación por teclado

Todos los elementos interactivos deben ser accesibles con teclado. WCAG 2.4.7 requiere un indicador de foco visible. Remover el outline sin reemplazarlo rompe la navegación para usuarios de teclado y tecnologías asistivas.

Usá Tab para avanzar · Shift+Tab para retroceder · Enter/Espacio para activar. Desactivá el focus para ver el impacto.

Preferencia de contacto
06

Texto alternativo

El atributo alt describe el contenido o función de una imagen para usuarios que no pueden verla. Buenas reglas: imágenes informativas describen su contenido; imágenes decorativas usan alt vacío; imágenes funcionales describen su acción.

Caso Alt incorrecto Alt correcto
Foto de producto (e-commerce)
Alt incorrecto alt="imagen"
Alt correcto alt="Zapatilla running blanca, suela azul, talle 42"
Logo (enlace al inicio)
Alt incorrecto alt="logo.png"
Alt correcto alt="1778 Studio — volver al inicio"
Imagen decorativa
Alt incorrecto alt="separador decorativo con líneas"
Alt correcto alt="" (vacío — el lector la omite)
Gráfico de datos
Alt incorrecto alt="gráfico"
Alt correcto alt="Gráfico de barras: ventas crecieron 40% en Q4 2024 vs Q4 2023"
07

Movimiento reducido

El movimiento excesivo puede causar náuseas o convulsiones en personas con trastornos vestibulares, migraña, epilepsia fotosensible o autismo. WCAG 2.3.3 (AAA) requiere una forma de detener animaciones no esenciales.

Reducir movimiento
Detección del sistema: Preferencia del sistema: normal
Media query CSS: 
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}

La accesibilidad no es una función — es el fundamento del buen diseño.
